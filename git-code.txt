1、在远程git网页上建一个新仓库+第一次把本地文件夹推到远程仓库
code README.md
git init
git add README.md  or  git add .
git commit -m "first commit"
git branch -M main  #默认master分支，重命名为main分支
git remote add origin https://github.com/csuenxi/-.git
git push -u origin main

git commit --amend -m "first commit"  #修改最近一次提交commit

--你创建 GitHub 仓库的时候勾选了：
✅ Add a README
✅ Add .gitignore
✅ Add LICENSE
这样 GitHub 会自动创建第一次 commit，需要如下改动，执行：

git pull origin main --allow-unrelated-histories

因为两个仓库没有共同祖先，所以需要：
--allow-unrelated-histories
如果没有冲突，再：

git push -u origin main

有冲突，自己改，再git add README.md等等push回去

2、命令	                   含义	                         是否推荐
git push origin main	 推送本地 main 到 origin/main	✅ 最常用
git push	             推送到当前分支关联的远程	     ✅ 建立 upstream 后推荐
git push -u origin main	 第一次推送并建立关联	         ✅ 第一次最推荐

本地分支（branch）：main
远程仓库（remote）：origin
远程分支（remote branch）：origin/main

3、查看当前有哪些远程仓库

git remote -v

例如：
origin  https://github.com/csuenxi/git-test.git (fetch)
origin  https://github.com/csuenxi/git-test.git (push)
说明当前远程仓库叫 origin
